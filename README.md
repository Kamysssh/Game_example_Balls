# 🎨 Игра про шарики

Интерактивная игра с красочными шариками, которые двигаются по экрану, сталкиваются и смешивают свои цвета!

## 📖 Описание

Это увлекательная игра-симуляция, где:
- **300 цветных шариков** свободно двигаются по игровому полю
- При **столкновении** шарики смешивают свои цвета (RGB смешивание)
- Вы можете **всасывать** шарики в инвентарь левой кнопкой мыши
- Можете **выплёвывать** их обратно правой кнопкой мыши
- Можете **перетаскивать** шарики из инвентаря на поле
- Есть **зона удаления** (красная корзина) для удаления ненужных шариков
- Красивая **3D графика** с объёмными бликами и тенями

### Особенности игры

- ⚡ **60 FPS** плавная анимация
- 🎨 **Реалистичное смешивание цветов** при столкновениях
- 🖱️ **Интуитивное управление** мышью и клавиатурой
- 📦 **Система инвентаря** для сбора шариков
- 🗑️ **Drag & Drop** для перемещения шариков
- ⏸️ **Пауза** и другие удобные функции

## 🎮 Управление

### Мышь
- **ЛКМ (левая кнопка)** - всосать шарик с поля в инвентарь (если курсор рядом с шариком)
- **ЛКМ на инвентаре** - взять шарик и перетащить его на поле или в корзину
- **ПКМ (правая кнопка)** - выплюнуть шарик из инвентаря в направлении курсора

### Клавиатура
- **SPACE** - пауза/продолжить игру
- **N** - добавить новый случайный шарик
- **R** - сброс игры (очистить и создать 300 новых шариков)
- **C** - очистить всё (удалить все шарики)
- **ESC** - выход из игры

## 🐳 Запуск в Docker

Docker-контейнер настроен для запуска игры с поддержкой графического интерфейса через X11.

### Для Linux

#### 1. Разрешите доступ к X-серверу

```bash
xhost +local:docker
```

#### 2. Соберите Docker-образ

```bash
docker build -t ball-game .
```

#### 3. Запустите контейнер

```bash
docker run --rm -it \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  ball-game
```

#### 4. После использования (опционально)

```bash
xhost -local:docker
```

### Для Windows (с WSL2 и VcXsrv/Xming)

#### 1. Установите X-сервер для Windows

Скачайте и установите один из:
- [VcXsrv](https://sourceforge.net/projects/vcxsrv/)
- [Xming](https://sourceforge.net/projects/xming/)

#### 2. Запустите X-сервер

Запустите VcXsrv/Xming с параметрами:
- Display number: 0
- "Disable access control" (отключите контроль доступа)

#### 3. В WSL2 установите переменную DISPLAY

```bash
export DISPLAY=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}'):0
```

#### 4. Соберите и запустите контейнер

```bash
docker build -t ball-game .
docker run --rm -it \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  ball-game
```

### Для macOS (с XQuartz)

#### 1. Установите XQuartz

```bash
brew install --cask xquartz
```

#### 2. Запустите XQuartz и настройте

Откройте XQuartz → Preferences → Security:
- ✅ Включите "Allow connections from network clients"

#### 3. Перезапустите XQuartz и разрешите подключения

```bash
xhost +localhost
```

#### 4. Соберите и запустите контейнер

```bash
docker build -t ball-game .
docker run --rm -it \
  -e DISPLAY=host.docker.internal:0 \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  ball-game
```

## 🎯 Как играть

1. **Наблюдайте** за движением шариков и смешиванием цветов
2. **Собирайте** интересные шарики в инвентарь левой кнопкой мыши
3. **Экспериментируйте** с выплёвыванием шариков правой кнопкой

6. **Создавайте** новые комбинации цветов!

## 🎨 Как работает смешивание цветов

При столкновении двух шариков их RGB-компоненты усредняются с учётом специального алгоритма:
- Математическое усреднение RGB-компонентов
- Нормализация для предотвращения "посерения" цветов
- Усиление насыщенности для комплементарных цветов
- Взвешивание по яркости для более интересных результатов

Например:
- Красный + Синий = Фиолетовый
- Жёлтый + Синий = Зелёный
- Красный + Зелёный = Коричневый/Оранжевый

## 🐛 Известные проблемы и решения

### Не запускается в Docker
- Убедитесь, что X-сервер запущен
- Проверьте, что переменная `DISPLAY` установлена правильно
- Для Linux: выполните `xhost +local:docker`

### Низкая производительность
- Уменьшите начальное количество шариков (в `game.py` измените `INITIAL_BALLS_COUNT`)
- Снизьте FPS (измените `FPS` в `game.py`)

### Ошибка "No available video device"
- В Docker нужен X11, убедитесь, что X-сервер доступен
- Проверьте монтирование `/tmp/.X11-unix`
